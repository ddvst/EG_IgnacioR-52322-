1. CSS (Cascading Style Sheets u Hojas de Estilo en Cascada) es un lenguaje informático (específicamente, de hojas de estilo) que cuenta con la funcionalidad de poder proporcionar un estilo a un documento HTML, permitiendo el ordenamiento de elementos de manera avanzada y el estilizado de cada etiqueta, incluyendo fuentes, colores, divisiones de espacio, tamaños, entre otros.

2. Las reglas en CSS son lo que modifican nuestro contenido, estas pueden ser (Como fue declarado en el punto anterior): Colores, fuentes, tamaño de texto, alineaciones, etc.

3. Las tres maneras de aplicar reglas del lenguaje CSS son las siguientes:
  a- Inline: Aplicamos las reglas directamente dentro de la etiqueta como un parámetro mas de la misma, utilizando la nomenclatura style="" que contiene dentro una regla CSS que corresponde al contenido de la misma. Esta es una práctica no recomendada, aunque puede resultar útil para realizar pruebas rápidas.
  b- Block: Declaramos todas las reglas CSS en la sección <head></head> de la página utilizando la etiqueta <style></style> dentro de ella. Aquí colocaremos todas las reglas que queremos aplicar en el documento. Esta práctica no es muy común.
  c- External File/Referencia/Link: Las reglas CSS se encontrarán en un archivo externo al HTML el cual será referenciado en el documento con una etiqueta <link> que cuente con el parámetro rel="stylesheet" y la dirección al archivo *.css.
4. Los selectores son los nombres por los cual referenciamos fragmentos del documento HTML, esto puede ser el documento en su enteridad (*), una etiqueta en específico (h1), una clase (.clase), una ID (#ID) o relaciones entre ellos, que pueden ser de adyacencia, de padre-hijo, hermanos, entre otros.
5. Una clase es una palabra clave añadida a un selector que específica una regla según una interacción o un estado que afecta a un elemento. Ejemplos populares son :hover (Ej: .clase:hover), que se activa al pasar el mouse por encima del elemento; :active (Ej: h1:active), que se activa al clickear sobre el elemento, entre otros.
6. La herencia es un proceso por el cual elementos que estan encerrados dentro de otro recibiran reglas relativas a las que tiene su denominado "padre". Por ejemplo, podemos tener un <h1> dentro de un <div>, siendo que el padre (<div>) contiene el color de texto azul (por ejemplo), entonces el <h1> podrá heredar este color. 
7. La razón por la que se refiere al lenguaje como un proceso en cascada es por la jerarquía con la que este cuenta; La prioridad avanza de mayor importancia a menor importancia: #ID > .Clases > Etiquetas.
